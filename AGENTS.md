# Project Agent Instructions

## Project overview

Artist portfolio site for Carla. Public pages (portfolio, artwork detail, category, about, contact) plus an authenticated admin dashboard for managing artworks, categories, and image uploads.

## Tech stack

- **Framework:** React 19, TypeScript (strict), Vite, TanStack Start + TanStack Router (file-based routing in `src/routes`)
- **Styling/UI:** Tailwind CSS v4, shadcn/ui components on Base UI primitives (`@base-ui/react`), lucide-react icons, sonner toasts
- **Auth & data:** better-auth (admin login), Postgres via Kysely + `pg`, raw SQL migrations in `migrations/`
- **Services:** Bunny.net storage/CDN (image hosting), Resend (contact form email), Cloudflare Turnstile (spam protection)
- **Tooling:** pnpm, Vitest + Testing Library (jsdom), oxlint + oxfmt

## Common commands

- `pnpm dev` ŌĆö dev server on port 3000
- `pnpm build` / `pnpm preview` ŌĆö production build / preview
- `pnpm test` ŌĆö Vitest
- `pnpm typecheck` ŌĆö `tsc --noEmit`
- `pnpm lint` / `pnpm format` ŌĆö oxlint / oxfmt
- `pnpm db:migrate` / `pnpm db:reset` ŌĆö apply / reset the database schema

## Project structure

- `src/routes/` ŌĆö file-based routes (`__root.tsx` is the root layout; includes `admin.tsx`, `login.tsx`, `api/auth/$.ts`)
- `src/components/` ŌĆö shared components; `components/ui/` is shadcn, `components/admin/` is the admin dashboard
- `src/lib/server/` ŌĆö server-only modules (`*.server.ts`: db, auth, Bunny, artworks, etc.)
- `src/lib/functions/` ŌĆö TanStack `createServerFn` wrappers called from the client
- `src/lib/client/` ŌĆö client-side helpers (e.g. auth client)
- `src/lib/shared/` ŌĆö types and utils shared across client/server (`cn` lives in `shared/utils.ts`)
- `src/data/` ŌĆö static data
- `migrations/`, `scripts/` ŌĆö SQL migrations and DB scripts

## Conventions & rules

- Use the `@/...` import alias only; never `#/...` imports or path aliases.
- Respect the `src/lib` layering: server-only code stays in `*.server.ts` files and must never be imported into client components; client-side data calls go through `src/lib/functions`.
- `src/routeTree.gen.ts` is generated by TanStack Router ŌĆö do not edit it by hand.
- After making changes, run `pnpm typecheck` and `pnpm lint`.
- Required environment variables live in `.env.local` (see `src/lib/server/env.server.ts` for the list); never commit secrets.

## shadcn/ui components

All shadcn-style components in `src/components/ui` must use Base UI primitives from `@base-ui/react`.

Do **not** use Radix primitives for shadcn components. Do **not** add new imports from `radix-ui` or `@radix-ui/*` in `src/components/ui`.

When adding or updating a shadcn component:

1. First verify the component is used in the project. If unused, ask whether to convert, remove, or defer it.
2. Obtain the official shadcn Base UI source for that component from the shadcn website/registry.
3. Use the official Base UI structure as the baseline.
4. Preserve this projectŌĆÖs local styling changes, variants, props, and wrapper APIs where possible.
5. Convert Radix-specific APIs to Base UI APIs, for example `asChild`/`onSelect` patterns may need Base UI `render`/`onClick` equivalents.
6. Run typecheck and lint after each component change.
